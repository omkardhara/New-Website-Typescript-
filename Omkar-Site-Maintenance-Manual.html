Part 1 · Orientation & Facts

This is a Next.js 14 (App Router, static export) + TypeScript site on Vercel. The code is compiled before going live, so a mistake in the wrong place can fail the build and block the deploy — but a failed build never takes the live site down; Vercel keeps serving the last good version. The whole game is "make the build pass."

📌 Your repo already documents itself — read these first

The project ships its own up-to-date guides. This manual consolidates and extends them; when in doubt, the in-repo docs + the live code win.

  • docs/site-editing-guide.md — step-by-step for routine content tasks (writing, work, video, press, timeline).
  • docs/image-sizing-guide.md — exact pixel sizes / max file sizes for every image slot.
  • CLAUDE.md, app/CLAUDE.md, components/CLAUDE.md, data/CLAUDE.md — architecture, routing, schemas, and hard rules.
🚫 The one rule that governs everything

All content lives in typed structures. If an edit doesn't match the shape in data/types.ts (missing required field, value outside an allowed list, stray comma, unclosed quote), the build fails and the change never ships. Always run npm run build locally before pushing (Part 3).

2.3 The mental model: data files vs components

  • Data files (data/*.ts) — structured lists (projects, writing, videos, press…). Low-risk to edit if you match the types. Most repeatable content. (Part 5)
  • Components / pages (components/*.tsx, app/**/page.tsx) — one-off copy baked into the UI: the hero headline & stat numbers, contact details, the About-page prose, button labels, SEO text. (Part 6)
⚠️ The trap

The in-repo editing guide says "you'll never touch app/ or components/" — true for routine content, but you asked to change everything. The hero copy, the 4 stat numbers, contact details, About/Press/Media prose, and all SEO text live in components/pages, not data. Part 6 covers those.

Part 3 · The Edit & Deploy Workflow

3.1 Two ways to edit

MethodGood forRisk
GitHub website (pencil-edit, commit)A one-word text fix.⚠️ No type-check before commit → a bad edit ships to a failed Vercel build. Trivial text only.
Local editor (VS Code) + npm run dev/buildAnything structural: new content, data edits, multi-file work.✅ You catch errors before pushing. Recommended.

3.2 Local setup (one-time)

  1. Install Node.js LTS (v18+) and VS Code.
  2. git pull first, then npm install (one time, and again whenever dependencies change).
  3. npm run dev → http://localhost:3000, hot-reloads on save.

3.3 The pre-push build gate (don't skip)

npm run dev is forgiving; npm run build is what Vercel actually runs and surfaces type errors. Run it before every push:

npm run build

Ends with ✓ Compiled successfully + a route list → safe to push. A red Type error: / Failed to compile → fix first (Parts 4 & 9). Also useful: npm run lint.

3.4 Deploy

  • Push to main → Vercel builds & deploys (1–3 min). Watch vercel.com → project → Deployments. Green = live, Red = build failed (log shows file + line).
  • A failed build never breaks the live site — the previous good deployment keeps serving.

3.5 Rollback

  • Vercel instant rollback: Deployments → last good one → Promote to Production (seconds, no code change).
  • Git revert: git revert HEAD && git push.
  • If a push is rejected as out-of-date: git pull origin main --rebase then push (per CLAUDE.md).

Part 4 · The TypeScript Rulebook — data/types.ts

Every item in every data file must match a shape declared in data/types.ts. Read it before editing; it lists required fields, optional ones (?), and fixed value-lists ("unions"). Need a new field on a type? Add it there first.

4.1 Reading a type

export type WorkItem = {
  id: number;                // required
  slug: string;
  cat: 'performance'|'brand'|'installation'|'activism'|'production'|'art'|'education';  // one of these
  image?: string;             // ? = OPTIONAL
};
  • No ? = required. Omit it → Property 'x' is missing.
  • A list of 'a' | 'b' = only those exact strings allowed.
  • Unique id per item; slug unique & URL-safe (lowercase-hyphen).

4.3 The errors you'll actually hit

Build says…Means / fix
Property 'stat' is missing…Omitted a required field → add it.
Type '"x"' is not assignable to '…'Value outside an allowed union → use a listed value.
',' expected / Unterminated stringMissing comma between items, or an unescaped quote (4.4).
Cannot find name / moduleTypo in import or filename case mismatch.

4.4 Quotes & apostrophes

// ✗ breaks:  excerpt: 'It doesn't work',
// ✓ escape: excerpt: 'It doesn\'t work',
// ✓ backticks (also for multi-line article/content bodies): excerpt: `It doesn't work`,

In JSX text (components/pages), write a literal apostrophe as &apos; (existing files do, e.g. Let&apos;s work) — ESLint flags a raw '.

Part 5 · Changing Content — Data Files

These mirror docs/site-editing-guide.md (your authoritative step-by-step). Golden method: copy a working entry, edit the copy, then npm run build.

5.1 A writing piece (two files)

Body in writing-content.ts, metadata in site.ts → NOTES:

// data/writing-content.ts — key MUST equal the note slug
'my-new-slug': `Full text here.

Blank line = new paragraph (or stanza for poems).`,

// data/site.ts — NOTES, in the right section (article / poem / short-story)
{ id: 40, slug: 'my-new-slug', title: '…', date: 'May 2026', read: '5 min',
  tag: 'Craft', type: 'article', excerpt: 'Hook line.',
  image: '/images/cover.jpg', content: WRITING_CONTENT['my-new-slug'] },

The first entry in each type group is the featured card. External pieces (set url) skip the internal page.

5.2 Work project

Copy a WORK entry; set unique id/slug, valid cat, the text fields, image, images[]+captions[], and article. New category → add to the cat union in types.ts and to WORK_FILTERS. External? add url. Video at top? add yt.

5.3 Red Bull note

In redbull-notes.ts: add with publication: 'Red Bull', a url, and image: '/images/redbull/<slug>.jpg' (drop that file into public/images/redbull/). These are external — no internal page.

5.4 Video

Copy a VIDEOS entry; set title/sub/yt/thumb (thumb = https://img.youtube.com/vi/<ID>/maxresdefault.jpg). portrait:true for vertical; startTime to begin mid-video. If a thumbnail won't load, use a local /images/… file (as the Doordarshan entry does).

5.5 Press item

Add the scan to public/images/press/, then copy a PRESS entry: unique id/slug, publication, title, year, type:'image', src, category. featured:true = larger card + appears on the About page.

5.6 Story / timeline · Offerings · Socials

Chapters in timeline.ts → CHAPTERS_FULL, events in TIMELINE (year/title/tag/desc + optional image). OFFERINGS and SOCIALS are plain arrays in site.ts — edit href/handle directly.

5.7 Testimonials

⚠️ Approval-gated

testimonials.ts entries carry an approved flag and the file is marked DRAFT until quotes are confirmed by the people quoted. Only flip approved: true once you have their sign-off, and keep context (internal note) accurate.

Part 6 · Changing Content — Components & Pages

For the one-off copy that isn't in data files. Edit the JSX text only; leave tags/classes/props intact.

6.1 Hero  components/Hero.tsx

Eyebrow, headline, aside, CTAs are JSX text. The four animated counters are the STATS array — change num/label/suffix (currently 12 Years · 500 Murals · 17 Press Features · 4000 km):

const STATS = [
  { num: 12, label: 'Years' },
  { num: 500, label: 'Murals' },
  { num: 17, label: 'Press Features' },
  { num: 4000, label: 'Rickshaw Run', suffix: ' km' },
];

The headline uses <br /> for line breaks and <em>…</em> for the serif italic. Hero portrait = replace public/images/portrait.jpg (4:5, see image guide).

6.2 Contact email & WhatsApp

Two places, both currently correct (919920499089): the constant WHATSAPP_NUMBER in components/ContactBand.tsx and the wa.me/mailto links in app/contact/page.tsx. Update both if the number/email changes (digits only, no +/spaces).

6.3 About / Press / Media page copy

Prose for these lives directly in components/AboutPage.tsx and the respective app/.../page.tsx. Per CLAUDE.md, one-off sections are intentionally inlined rather than split into new components — edit the JSX in place.

6.4 SEO & social previews

Global: app/layout.tsx → metadata (browser-tab title, description, keywords, OG/Twitter image /og-2025.jpg) and the personSchema JSON-LD. Per page: each route exports generateMetadata() with its own title/description/canonical/OG — follow the existing pattern (also in app/CLAUDE.md). Article pages additionally emit content + BreadcrumbList JSON-LD.

🚫 OG image: never reuse /og-image.jpg

The social-share image was deliberately renamed to /og-2025.jpg to bust WhatsApp's aggressive preview cache. To change it, replace public/og-2025.jpg or add a new filename and update the references in layout.tsx — but don't revert to the old og-image.jpg path.

6.5 Colours & fonts

Colour tokens are the :root CSS variables at the top of app/globals.css; fonts in lib/fonts.ts. Change a hex and it cascades everywhere.

📌 Misleading name

--gold is actually green (#2E6B4F) — the primary accent. --ember is the orange (#C4621D). Don't be fooled by the variable name.

Part 7 · Adding New Content & Images

Everything in Part 5/6 doubles as "add" instructions — copy a working item. This part collects the image rules, since nearly every addition involves a photo.

7.1 Image rules

RuleDetail
LocationLocal images → public/. A file at public/images/x.jpg is referenced as /images/x.jpg (no public).
SizesFollow docs/image-sizing-guide.md — it gives exact px + max-KB per slot (hero 960×1200 ≤300KB; work thumb 800×600 ≤200KB; writing cover 16:9 ≤250KB; OG 1200×630 ≤300KB; etc.).
Naminglowercase-hyphen, no spaces (rickshaw-run-finish.jpg). JPG for photos, PNG only for transparency.
Case-sensitiveVercel builds on Linux = case-sensitive. Photo.JPG ≠ photo.jpg. Path in code must match the file exactly, extension included.
Remote imagesAny non-local host must be in remotePatterns in next.config.mjs (YouTube already is).
🚫 "Works locally, breaks on Vercel"

Almost always filename casing or a wrong extension (.jpg vs .jpeg vs .png). Windows ignores case; Vercel doesn't. If an image 404s only in production, check the exact name.

7.2 Adding a whole new page or section

  • New route: create app/<name>/page.tsx with a default-export component and a generateMetadata() (copy app/contact/page.tsx as a template). Live at /<name>.
  • New homepage section: per CLAUDE.md, inline it in app/page.tsx rather than making a one-off component. Styling via globals.css / inline styles (no Tailwind).
  • Interactivity/hooks (useState, onClick…): the file's first line must be 'use client'.

Part 8 · Rules You Must Not Break

RuleWhy
Don't increase homepage tab preview counts (3 Work, 3 Media, 3 per writing section). They're PREVIEW_COUNT in each components/tabs/*Tab.tsx.Intentional design. Adding cards updates the "See all N →" count automatically; the homepage still shows 3.
Don't add Tailwind classes.Tailwind is vestigial/unused; all styling is globals.css + inline. Utility classes silently do nothing.

Part 9 · Troubleshooting

9.1 Vercel build failed (red X)

  1. Open the failed deployment → read the log; it names the file, line, reason.
  2. Reproduce locally with npm run build.
  3. Usual causes: type mismatch / missing required field (Part 4.3), unescaped quote (4.4), image path/case (7.1).
  4. Fix → build passes → push. Live site was untouched throughout.

9.2 "I pushed but nothing changed"

  • Did the Vercel build go green? Red = your change didn't ship.
  • On main? (git status, git log.)
  • Hard-refresh (Ctrl/Cmd+Shift+R) or incognito — CDN/browser cache.

9.3 An image won't load

  • Only in production? → filename casing/extension (7.1).
  • Path starts /images/… (no public)? Remote host whitelisted in next.config.mjs? File actually committed (git status)?

9.4 A page 404s or content is missing

  • Work/Press/Writing: the slug in the URL must match the data entry, and the entry must not have a url (external items don't generate internal pages).
  • New writing piece not on its list? Check type/publication — they decide the section (essays vs redbull vs poems).
  • New page route: the folder must contain page.tsx exactly.

9.5 Build error about hooks / "use client"

useState/useEffect/onClick only work in client components → add 'use client' as the file's first line.

9.6 Local dev won't start

  • command not found → Node not installed / reopen terminal.
  • Module errors after git pull → re-run npm install.
  • Port busy → npm run dev -- -p 3001.

9.7 Emergency rollback

Vercel → Deployments → last good → Promote to Production. Or git revert HEAD && git push.
